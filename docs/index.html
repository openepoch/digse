digse

Dig Search Engines — a lightweight, privacy-focused metasearch engine CLI in Rust.

Install

One command. No build step. No admin or sudo required.

curl -fsSL https://raw.githubusercontent.com/openepoch/digse/main/docs/install.sh | sh

Installs digse into ~/.local/bin. view the script · source

irm https://raw.githubusercontent.com/openepoch/digse/main/docs/install.ps1 | iex

Installs digse into %LOCALAPPDATA%\digse and adds it to your user PATH. view the script · source

Install a specific version: set VERSION=v0.2.0 (sh) or $env:DIGSE_VERSION="v0.2.0" (PowerShell) before the pipe.

Run it

digse version          # digse 0.0.1  (target: linux-x86_64)
digse start            # start the local daemon at http://127.0.0.1:8888
digse ps               # is it running? (prints pid + url)
digse stop             # stop it
digse startup add      # start at boot (systemd on Linux, Run key on Windows)

Then open http://127.0.0.1:8888 for the search UI, or query /search?q=... for JSON. digse startup add needs no administrator privileges on any platform.

All Commands

Server Control

digse start [--host HOST] [--port PORT]    # Start daemon (default: 127.0.0.1:8888)
digse restart                                # Restart or start if stopped
digse stop                                   # Stop running daemon
digse ps                                     # Show status (pid + URL)

Version & Info

digse version                               # Print version and build target
digse --version                              # Short form (just version)

Boot Autostart

digse startup status                        # Show autostart status
digse startup add                           # Enable boot autostart (no admin needed)
digse startup remove                        # Disable autostart

Linux uses systemd user service, Windows uses registry Run key.

Configuration

digse config path                           # Print config file path (~/.digse/config.toml)
digse config show                            # Print full configuration as TOML
digse config init                            # Write default config if missing
digse config get serve.port                  # Get a config value by dotted key
digse config set serve.port 9999            # Set and persist a config value

Engines & Categories

digse list engines                          # List all search engines (206 total)
digse list categories                       # List engine categories (12 total)

Uninstall

digse uninstall                             # Remove binary, config, and autostart

The uninstall command removes the installed binary, configuration directory, and autostart entries, but does not affect the local repository if you built from source.

Global Options

-v, -vv, -vvv, -vvvv                       # Increase verbosity
--quiet                                      # Minimal output
-h, --help                                  # Print help
-V, --version                               # Print version

Features

  • 206 engines across 12 categories
  • Structured JSON output, always
  • Local HTTP API + HTML search UI
  • Persisted config (~/.digse/config.toml)
  • 14 result types (web, images, videos…)
  • URL-based filetype/include/exclude filters
  • Engine selection, concurrency, rate limiting
  • Cross-platform daemon (Linux, Windows)
  • Boot autostart, no admin
  • Usable as a Rust library
